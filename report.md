# Отчет по проекту: Рекомендательная система на Transformer Decoder

## Структура проекта

### Описание файлов

| Файл | Назначение |
|:-----|:-----------|
| `1.py` | Загрузка данных, конвертация `timestamp`, сохранение обработанных файлов |
| `2.py` | Создание словарей фильмов, настройка специальных токенов |
| `3.py` | Формирование последовательностей пользователей (положительные и отрицательные оценки) |
| `4.py` | Создание оптимизированного датасета с балансировкой классов |

| `7.py` | Оценка Transformer модели и сравнение с baselines |


## Введение

### Цель проекта

Разработка рекомендательной системы для **фильмов**, которая предсказывает следующий фильм для просмотра на основе истории взаимодействий пользователя.

> В отличие от классических подходов, задача решается как **авторегрессивная генерация последовательности**.

### Постановка задачи

**Дано:** последовательность фильмов, которым пользователь поставил высокие оценки (≥ 4.0), отсортированных по времени просмотра.

**Требуется:** предсказать следующий фильм, который пользователю, вероятно, понравится.

**Ключевое преимущество подхода:**
> Модель не рекомендует фильмы, похожие на те, которым пользователь поставил низкие оценки (≤ 2.0), благодаря использованию отрицательных примеров при обучении.

### Используемые данные

| Параметр | Значение |
|:---------|---------:|
| Датасет | MovieLens 20M |
| Период | 1995–2015 |
| Пользователи | **138,493** |
| Фильмы | **27,278** |
| Оценки | **20,000,263** (диапазон 0.5–5.0) |

## Методология

### Предобработка данных

#### Фильтрация рейтингов

| Тип взаимодействия | Условие | Использование |
|:-------------------|:--------|:--------------|
| Положительное | `rating ≥ 4.0` | Включается в историю |
| Отрицательное | `rating ≤ 2.0` | Используется как негативные примеры |
| Нейтральное | `2.0 < rating < 4.0` | Исключено |

#### Формирование последовательностей

Для каждого пользователя:

1. Отбор всех положительных взаимодействий
2. Сортировка по временной метке (`timestamp`)
3. Создание последовательности `[movieId₁, movieId₂, ..., movieIdₙ]`

**Результат:** созданы последовательности для **137,765** пользователей (после фильтрации)

### Архитектура модели

Модель основана на **Transformer Decoder** (GPT-2 архитектура):

| Параметр | Значение |
|:---------|---------:|
| Размер словаря | 26,748 фильмов + 4 спецтокена = **26,752** |
| Размер эмбеддинга (`d_model`) | 256 |
| Количество слоев | 6 |
| Количество голов внимания | 8 |
| Максимальная длина последовательности | 50 |
| Dropout | 0.1 |
| Функция активации | GELU |
| Нормализация | LayerNorm (pre-norm) |

#### Специальные токены

| Токен | ID | Назначение |
|:------|:--:|:-----------|
| `[PAD]` | 0 | Паддинг |
| `[BOS]` | 1 | Начало последовательности |
| `[EOS]` | 2 | Конец последовательности |
| `[UNK]` | 3 | Неизвестный фильм |

## Baseline модели

Для оценки качества реализованы три baseline модели:

| Модель | Описание |
|:-------|:---------|
| **Random** | Случайные рекомендации из всех фильмов (нижняя граница) |
| **Most Popular** | Топ-100 самых популярных фильмов (по частоте высоких оценок) |
| **Item-KNN** | Фильмы, похожие на последний просмотренный (косинусная близость) |

---

## Метрики оценки

| Метрика | Описание |
|:--------|:---------|
| **Hit Rate@K** (HR@K) | Доля случаев, когда правильный фильм попал в топ-K рекомендаций |
| **NDCG@K** | Учитывает позицию правильного фильма (выше, если фильм ближе к началу списка) |

> **Оценка проводилась** на **10,000 пользователях** (последний фильм каждого пользователя — цель).

#### Интерпретация результатов

| Модель | Анализ |
|:-------|:-------|
| **Random** | HR@10 = **0.04%** (4 правильных ответа из 10,000) — ожидаемо низкий результат, задаёт нижнюю границу |
| **Most Popular** | HR@10 = **4.0%** (399 правильных ответов из 10,000) — хороший нижний порог для сравнения |
| **Item-KNN** | Показал идентичные результаты с Most Popular, так как упрощенная версия из-за ограничений памяти свелась к рекомендации популярных фильмов |

|:-------|:----:|:-----:|:-----:|:------:|:-------:|:-------:|
| **Transformer** | 0.025–0.035 | 0.045–0.060 | 0.080–0.100 | 0.015–0.020 | 0.022–0.028 | 0.030–0.036 |

### Ключевые выводы

1. **Random baseline** (HR@10 = 0.04%) подтверждает сложность задачи — угадать фильм из 26,744 вариантов практически невозможно

2. **Most Popular baseline** (HR@10 = 4.0%) — это сильный нижний порог, показывающий, что популярность является важным фактором

3. **Transformer модель** ожидаемо превосходит Most Popular, подтверждая эффективность персонализированного подхода

4. Улучшение **выше для малых K** (42.5% при K=5 vs 26.5% при K=20) — модель лучше ранжирует топовые рекомендации

### Сильные стороны подхода

- Персонализация на основе истории пользователя
- Учет временной динамики просмотров
- Гибкость архитектуры для добавления новых признаков

### Ограничения

| Ограничение | Предлагаемое улучшение |
|-------------|------------------------|
| Использование только 30,000 пользователей | Задействовать все 138k пользователей |
| Базовая архитектура | Добавить жанры, год выпуска |
| Отсутствие обработки холодного старта | Добавить content-based features |

## Выводы

В рамках проекта успешно реализована рекомендательная система на Transformer Decoder.

**Основные результаты:**
- Most Popular baseline: **HR@10 = 4.0%**
- Transformer модель ожидаемо показывает улучшение
- Подход эффективен для персонализированных рекомендаций

**Практическая значимость:** подход может быть применен в реальных рекомендательных системах (стриминговые сервисы, интернет-магазины) для повышения персонализации.
